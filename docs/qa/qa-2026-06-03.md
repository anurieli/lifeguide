## 🟡 Secondary findings

0. **Core VoiceField (mic) — UX bugs (found live by user, `components/voice/VoiceField.tsx`).**
   - **Prompt overlap:** the recording card shows up to **3** AI prompts at once (`prompts.slice(0, 3)`), pinned to fixed corners (`PROMPT_ANCHORS`). They overlap the field's question text and the live transcript (see user screenshot — "Values to Live By" with three prompts colliding over the words).
   - **No discoverable way to end:** finishing the recording is done by **clicking the waveform** (`onClick … finish`, tooltip "Tap to finish" / "tap the wave when you're done"). There is no explicit button; users don't realize the waveform is the stop control.
   - **Requested change:** show **one** contextual prompt at a time (it should relate to what's being said — `voice.prompts` already receives the live `partial`, so the data is there; the UI just needs to render a single, non-overlapping prompt) and add a clear finish control. The question stays inside the field box. (Deferred — not yet implemented.)

1. **Vitest picks up worktree duplicates.** `npm test` runs tests from `.claude/worktrees/voice-field/tests/` alongside `tests/`, double-counting several files (ai-config, assemble, geometry, edges, distill-parse). Not a product bug; tighten `vitest.config.ts` include/exclude so worktrees aren't collected.
2. **Core field example placeholders contain personal-sounding content** (e.g. "over the last week I was building Lifeguide… coffee chat with a friend…"). They're placeholders, but read as real data; consider neutral examples.
3. **Settings AI section fails silently** (see Critical). Even after deploy, worth a visible empty/error state instead of a blank model list when `aiKeys.nodes` is unavailable.

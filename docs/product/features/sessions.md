# Sessions (the living entry) — surfaced as **Thoughts**

**Status:** built (v3, 2026-07-12) · **Element of:** Sessions (feeds the Core) · **Owns:** `sessions` (container rows); members are `captures` rows via `captures.sessionId`

> A session is one living journal entry: a brain dump you keep adding to. Tap ➕ and let it out; the entry can be reopened tomorrow to speak more, type into, drop photos or links on. Every piece stays a raw, immutable capture; the entry is the document view over them. The person can name the entry; the AI keeps a living description current for the list and for any agent traversing entries. **This is the single capture surface of the app** (v3, ADR 0010; the old flat-stream tab merged into it). The tab is labeled **Thoughts** (booklet icon — a session is a booklet; renamed from the short-lived "Dumps", v3.1 same day) and the main action exists on desktop as well as the phone: the phone's ➕, and on desktop a **scribbler pen** whose hover says **"Think out loud"**.

## 1. Purpose

The observation contract says: let the thought out first, structure later. The Thought Stream catches single thoughts; a session catches a *sitting*, the twenty-minute walk where one thread unspools across voice, a typed line, and a photo of the whiteboard. The roadmap's MVP capture spine names this entity ("Session: raw capture + metadata"). Because members stay ordinary captures with their raw artifacts kept forever, every future pass (decomposition into atomic thoughts, cross-session linking, pattern mining) can run retroactively over the whole archive; the session is the unit those passes will read.

## 2. User-facing behavior

**Starting one.** The main action creates a fresh session and lands inside its empty document. On the **phone** (bottom bar's big center ➕) it is already recording (no separate recording screen) — and the mic arms **before** the server answers with the new entry: the take's target is a promise the provider binds when creation resolves, so the first words never wait on the round-trip. On **desktop** (the pen in the rail below the nav items, and a pen in the list header — both titled "Think out loud") a plain click opens the document ready to type, with the mic one click away — auto-arming the mic on a mere click would throw a permission prompt unprompted. Tapping ➕ always starts a NEW entry — with one exception: from **inside an entry that is still empty** (no captures, no take saving into it), the main action reuses that entry instead of spawning an indistinguishable twin (on the phone it starts recording there; on desktop it's a no-op — you're already in a fresh page). Appending to an old entry goes through the list. If the mic is denied, a small note says "Mic unavailable. Tap the page and type." and the document stays fully usable. A too-short take (~1s) saves nothing.

**Quick record (press ➕, swipe up).** Pressing the main action and swiping up — on the phone's ➕, the desktop rail pen, or the list-header pen — starts recording **the instant the gesture commits**, finger still down, on any device (the swipe is an explicit ask to record, so the desktop permission-prompt rule doesn't apply). The entry is created and its document navigated to in the background while the take is already rolling. Inside an empty entry the gesture records right there — no new note. If entry creation fails (offline), the seconds-old take is dropped and the mic released. The gesture lives in `hooks/usePressSwipeUp.ts` (pointer events, ~36px upward travel, the post-swipe synthetic click suppressed).

**The entry (one continuous document).** The session's captures render in order as one flowing page: spoken passages as text (with a quiet inline player; "Listening back…" while transcribing; on failure, "the recording is safe" + Try again), typed passages, photos, pasted links (a typed paragraph that is a single bare URL commits as a link capture, exactly as the old stream composer treated it; it renders as a tappable link and ingest reads the page). There is no composer strip and no send button: tap anywhere on the page and type; the paragraph commits as a text capture on blur (Cmd/Ctrl+Enter also commits). On desktop the mic (primary) and photo controls are a **horizontal pair that rides the text caret**: click anywhere and the two buttons sit just right of where you're writing — the record button is literally at the cursor, one click from the thought (`components/sessions/caret.ts` measures the caret via the mirror-div technique; the pair tracks focus, clicks, arrow keys, and typing, clamped to the page column). With no caret anywhere, the pair rests in flow **below the last text/item** in the document. While a take is live the pair swaps for the slim pill at the same spot. The document is pure capture on desktop: the Coach window and its floating buttons **step aside entirely** while an entry is open and return the moment you leave (2026-07-13, settled after two earlier layouts — corner-stacked, then side-by-side with the Coach — both by Ariel's live feedback). On the phone the controls keep floating bottom-right (no caret to ride). A live take shows in-flow as a pulsing timer while the rest of the page stays usable, so you can type and drop photos mid-recording. While a take is live the mic swaps for one slim pill: discard (✕, two-tap "Sure?" confirm) · pause/resume (the timer freezes; paused stretches add no audio and no time) · save (■). **Saving is optimistic:** the moment you tap save, the take appears in the document as a pending row (gold dot · duration · "Processing…", the same shape as the live row it replaces) while the upload and capture write run behind it; the real capture row takes its place when the server has it, then "Listening back…" until transcription lands. The mic is free again immediately — the next take never waits on the previous one's upload. A small "What were you doing?" field holds context. Entries are never closed; reopening one days later and appending is the intended use.

**Name and living description (v3).** The header's title is an editable field: type a name and it is yours — `titleEditedAt` marks it person-entered and the AI never overwrites it (clearing the field hands naming back to the AI). Left blank, the digest titles the entry for the list. Beneath the header sits the **living description** (`summary`): re-synthesized as content lands (the debounced digest off each member's ingest) and additionally refreshed on **every open and leave** of the document (`sessions.refreshDigest`, which skips when the digest already covers the latest content, so repeated visits cost nothing). The description is the canonical line an agent pulls when traversing entries.

**Recording survives leaving the screen.** The live take belongs to the shell (`RecordingProvider`), not the document: navigate to Today, the Board, or the list mid-take and it keeps recording. Anywhere outside the live entry, a quiet pill floats top-center (gold dot · elapsed · "Recording"/"Paused"); tapping it returns to the entry. Starting a new take anywhere (➕, or the mic in another entry) saves the current one into its own session first; there is only ever one live take.

**The list.** The **Thoughts** tab is the archive: pinned entries first, then newest by activity; relative time, a pin glyph, voice/photo count glyphs, the name (person-entered or AI), and the one-sentence living description (until the digest runs, the entry's first words). Tap to open. Swipe left to pin/unpin. Swipe right to delete: the row parks open with a Delete button, a second tap confirms. "Select" enters multi-select; picking 2+ entries shows "Merge N into one". The header carries its own pen ("Think out loud") so a fresh entry never requires leaving the surface.

**Demo thoughts.** The empty state offers "Show me two example thoughts", and a quiet "Add demo thoughts" link sits under a populated list. It seeds two fully packed entries — voice takes (with transcript and a playable file), photos, typed passages — so the surface can be seen lived-in. The media is painted client-side (`components/sessions/demoMedia.ts`: a synthesized soft-tone WAV via `lib/demoWav.ts`, canvas-drawn sunrise/notebook photos) and uploaded like any capture file; `sessions.seedDemo` then inserts ordinary sessions/captures with extraction already done and the digest stamped, so no ingest runs and no model is called over demo text. Demo rows are real data: open them, append, or swipe them away. Their `sourceMeta` carries `demo: true`.

**Merging.** Merge folds the selected sessions into the earliest-started one: every element keeps its own timestamp, so the merged document reads in the chronological order each piece was added, interleaved across the original entries. A person-entered name survives the merge (the earliest such session's name wins); AI title/summary are cleared and the digest re-synthesizes over the combined content.

**Deleting.** Deleting removes the entry from the archive view only: the container row is deleted, member captures are soft-deleted (`isActive: false`) with their raw artifacts and `sessionId` intact, so nothing raw is ever lost and future retroactive passes still see what belonged together.

**Desktop.** Thoughts is a rail tab (booklet icon, `NotebookPen`) and the rail carries the pen main action (`PenLine`, hover "Think out loud"); the same document behavior applies, and pin/delete appear as hover actions on list rows (no swipe). Desktop remains the command center. The phone bar is five even slots, Today · Board · ➕ · Thoughts · account, with the ➕ the only dark element (active tabs get a light tint, never a solid block); Core, the Talk/Listener tab, and the Atmosphere music player (the audio engine itself, not just the orb) are desktop-only; the phone is capture-first. The separate **Thoughts** tab is gone on every device (ADR 0010): the flat stream surface merged into this one, and its component (`components/thoughts/ThoughtStream.tsx`) stays in the codebase unrouted, like the Brain Dump Lab before it.

## 3. Functions / actions

| Action | Trigger | What it does | Data touched |
|---|---|---|---|
| Start an entry | Tap ➕ (phone bar) / the pen (desktop rail + list header) | `sessions.create`, opens the empty document; on the phone the take (owned by `RecordingProvider`) arms against the creation promise, before the server answers; desktop opens ready to type, mic one click away. From inside a still-empty entry, reuses it instead of creating. Save uploads the whole file as the first capture (`sourceMeta` carries device, durationMs, recordingStartedAt) | `sessions` (create), `captures` (create), `_storage` |
| Quick record | Press ➕/pen + **swipe up** (any device) | Mic arms the instant the gesture commits (`usePressSwipeUp`); `sessions.create` + navigation happen behind the live take; inside a still-empty entry it records there instead of creating | `sessions` (create), then as voice append |
| Optimistic take save | Saving a take (■) | The take renders in its entry immediately (`pendingTakes`) while the upload + `captures.create` run fire-and-forget; the mic frees at once; a failure moves the take to `failedTakes` (retry never re-records) | `captures`, `_storage` |
| Append (voice/text/link/photo) | In the document: tap-and-type (commits on blur; a bare URL commits as a link capture), floating mic, floating photo | `captures.create` with `sessionId` (ownership-checked); bumps `sessions.updatedAt` | `captures`, `sessions.updatedAt` |
| Name the entry | The editable title field in the document header | `sessions.setTitle`: trims to 80 chars, stamps `titleEditedAt` (the digest then never writes title); clearing hands naming back to the AI | `sessions.title/titleEditedAt/updatedAt` |
| Refresh the digest | Opening the document, leaving it | `sessions.refreshDigest`: schedules a digest run unless the digest already covers the latest content (or the entry has no active members) | `sessions.title/summary/digest` (via the digest run) |
| Pause / resume a take | The live pill's ⏸/▶ | `MediaRecorder.pause()/resume()`; paused time adds no audio and no elapsed | none (in-memory) |
| Discard a take | The live pill's ✕, two-tap confirm | Stops and throws the audio away; the entry stays open (husk rules apply if it stays empty) | none (in-memory) |
| Background take | Navigating anywhere mid-take | Recording continues (provider outlives the document); a top-center pill shows elapsed and returns to the entry | none until save |
| Pin / unpin | Swipe left (phone), hover pin (desktop) | `sessions.setPinned`; pinned entries lead the list | `sessions.pinnedAt` |
| Delete | Swipe right + confirm tap (phone), hover trash + confirm (desktop) | `sessions.remove`: deletes the container, soft-deletes members (raw kept, `sessionId` kept) | `sessions` (delete), `captures.isActive` |
| Merge | Select 2+ in the list → Merge | `sessions.merge`: re-parents every member onto the earliest-started session, deletes the emptied rows, clears title/summary, re-digests | `sessions`, `captures.sessionId` |
| Visit stamp | Opening a document | `sessions.touchOpened` writes `lastOpenedAt` (no `updatedAt` bump; reading must not resort the list) | `sessions.lastOpenedAt` |
| Ingest + distill members | Auto | Unchanged Thought Stream pipeline per capture | `captures.extractedText/extraction/distilled` |
| Seed demo thoughts | Empty state / list footer | Client paints + uploads demo media, `sessions.seedDemo` inserts two packed entries (extraction done, digest stamped; no AI calls) | `sessions`, `captures`, `_storage` |
| Digest | ~30s after a member's ingest completes; also via Refresh | AI writes `title` + `summary` onto the session (idempotent; last append wins; skipped while any member is still ingesting; **never overwrites a person-entered title**) | `sessions.title/summary/digest` |
| Set context | "What were you doing?" field | Saves `doing` (≤200 chars) | `sessions.doing` |
| List / open | Sessions tab | `sessions.list` (derived preview + counts) / `sessions.get` (ordered members, file URLs resolved) | read-only |
| Retry transcription | Entry's Try again | Existing `captures.reprocess`; digest refreshes after | `captures.extraction`, then as ingest |
| No husks | Back action in an entry; sweep when the list renders | `sessions.deleteIfEmpty` removes a container with no active members; the server re-checks emptiness, so it never races an append (raw captures are never touched; an empty container holds none). Exempt from both paths: the entry a live take is recording into, and any entry a finished take is still saving (or failed to save) into | `sessions` (delete) |

## 4. Dynamics with other elements

- **Thought Stream.** Merged into this surface as of v3 (ADR 0010): the flat stream is no longer a nav tab anywhere. Its **pipeline is unchanged and still load-bearing** — every member capture goes through the same ingest (transcription, link extraction, image reading) and distillation, and loose captures (`sessionId` absent: board intake, `voice.brainDump`) still work and surface in the board Inbox. See [`thought-stream.md`](thought-stream.md) for the pipeline; the stream *view* is retired pending a possible return as a filter/inspector over all captures.
- **Vision Board.** Unchanged. Session members surface in the board Inbox like any unplaced capture; a photo added to an entry from the phone is also inspiration waiting to be placed.
- **Journal (proposed).** The Journal's morning/night beats will *open* sessions (the roadmap's three front doors); the Journal owns `prompts` and writes into this table when built. See [`journal.md`](journal.md).
- **The Center / Core.** Not yet wired; sessions are upstream signal, like the stream.
- **The Listener.** Separate for now (`interviewSessions`); unifying Listener calls into this archive is an open question (see the listener memory backbone research note).

## 5. States and edge cases

- **No AI keys:** entries and captures land; digest no-ops with fallback title; transcription errors show with retry; nothing is lost.
- **Failed save of a take (network):** the finished blob is kept in memory (by the provider, so it survives leaving the document) and the entry shows "That take didn't save; it's still here" with Try again, so retry never re-records. Saves being fire-and-forget, several takes can fail independently; they accumulate (`failedTakes`) and one Try again retries them all. Killing the app mid-take still loses it (accepted; crash-recovery buffer is a parked follow-up).
- **Entry creation fails under quick record (offline):** the just-armed take has nowhere to land; the provider cancels it and releases the mic. At most a couple of seconds of audio are lost, at the very start of a sitting.
- **Take saving while its entry looks empty:** an entry whose only content is a take still uploading (or one that failed and awaits retry) is not a husk — the back action and the list sweep both leave it alone until the save resolves.
- **Mic dies mid-session (or never starts):** a synchronous `MediaRecorder.start()` failure is caught and surfaces as the mic-unavailable note instead of an unhandled error; the take target clears so no zombie "Recording" pill survives.
- **Failed transcription:** the audio file is stored regardless; the entry says "the recording is safe" with Try again.
- **Typed text pending on leave:** the back action commits a non-empty draft as a capture instead of husk-checking; a failed commit puts the text back on the page.
- **Empty session:** the ➕ flow creates the container before any content, so bailing out immediately is the common path; `deleteIfEmpty` clears it on the entry's back action, and the list sweeps any zero-count row on render.
- **Swipe vs tap vs scroll:** rows use `touch-action: pan-y` (vertical scroll stays native); a horizontal swipe suppresses the click that browsers fire after touchend, so a swipe never opens the entry.
- **Digest burst:** several appends in a row cost one model call (each ingest completion schedules a run 30s out; runs skip while any member is pending, and overwrite idempotently).
- **iOS Safari:** `useAudioRecorder` negotiates audio/mp4; ingest maps the extension. On-phone QA tracked in `TO-CHECK.md`.

## 6. AI involvement

One new node, `sessionDigest` (`convex/ai/config.ts`): OpenRouter, JSON `{title, summary}`, grounded strictly in member text via `lib/sessionDigest.ts` (`assembleDigestInput`, 6k cap, chronological, labeled by kind). Members' transcription/vision/distillation are the existing Thought Stream nodes.

## 7. Data touched

`sessions { userId, title?, titleEditedAt?, summary?, doing?, device: phone|desktop, digest?{status: pending|done|error, at?}, startedAt, updatedAt, pinnedAt?, lastOpenedAt? }` (index `by_user_updated`); `captures.sessionId?` (index `by_session [sessionId, createdAt]`). `titleEditedAt` marks the name person-entered; the digest write guards on it. Element-level metadata rides on each capture: `createdAt` (exact add time), and for audio `sourceMeta` JSON carries `device`, `durationMs`, `recordingStartedAt`. See [`../../architecture/data-model.md`](../../architecture/data-model.md) and [ADR 0008](../../decisions/0008-sessions-as-container-over-captures.md).
